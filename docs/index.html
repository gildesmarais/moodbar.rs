MOODBAR.RS / CROSS-PLATFORM COMEBACK

Music visualization toolkit for the moodbar format.

Moodbar is back with a cross-platform approach: one Rust analysis core, multiple output targets (CLI today, WASM now, native bindings next). The original moodbar idea from the Amarok era returns as a modern, composable toolkit.

Try Browser Demo View Source Read WASM Plan

Core Promise

Decode and analyze once. Render everywhere: raw RGB bytes, SVG, PNG, and JS bindings.

Audience

Audio developers and OSS contributors building visual music tools across platforms.

Compatibility

Preserves legacy raw moodbar output while extending into modern rendering and APIs.

Quick Start (CLI)

cargo run -p moodbar -- generate -i input.ogg -o output.svg --format svg

Quick Start (WASM Demo)

make wasm
python3 -m http.server
# open http://localhost:8000/docs/wasm-demo.html
# for GitHub Pages deployment assets
make wasm-docs

Architecture

  • crates/moodbar-core: DSP analysis, normalization, rendering primitives.
  • crates/moodbar-cli: automation-friendly commands for generate/batch/inspect.
  • crates/moodbar-wasm: browser/Node bindings via wasm-bindgen.

Back when Amarok painted songs as color timelines, moodbars felt like musical fingerprints. This project keeps that spirit, but makes it portable and hackable.
